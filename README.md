# Support

Welcome to the support repository for The Accord Project.

If you need help, have found a bug, want to request a feature, or need to report something that is not working as expected, please submit a GitHub issue so our team can review and track it properly.

## Get Support

Submit a support request here:

**Issues:** https://github.com/theaccordproject/support/issues

From that page:

1. Click **New issue**
2. Enter a clear title
3. Add the details of your request
4. Submit the issue

## When to Open an Issue

Please open an issue for things like:

- Bug reports
- Setup or configuration problems
- Product questions
- Documentation errors or missing steps
- Feature requests
- Unexpected behavior in the platform

## Before You Submit

Before opening a new issue, please:

- Check existing issues to see whether it has already been reported
- Gather any screenshots, error messages, or steps to reproduce the issue
- Make sure your title clearly describes the problem

## What to Include

The more specific you are, the faster we can help.

### For bugs
Include:

- What you expected to happen
- What actually happened
- Steps to reproduce the issue
- Screenshots, logs, or error messages
- Browser, device, or operating system details
- Product area affected, if known

### For support questions
Include:

- What you are trying to do
- Where you got stuck
- Any screenshots or error messages
- Any steps you have already tried

### For feature requests
Include:

- The problem you want to solve
- Why it matters
- Your suggested solution, if you have one

## Suggested Issue Title Format

Use a title that is short and specific.

Examples:

- Bug: Donation form fails after submit
- Support: Need help connecting payment account
- Docs: Setup instructions missing environment variable step
- Feature Request: Add recurring donation receipt settings

## Helpful Details to Add

If relevant, include:

- Product name: AccordPAY, TapONE, TapAPP, or TapBOOK
- Environment: production, staging, or local
- Date and time the issue occurred
- Screenshots or screen recordings
- Related links or issue references

## Please Do Not Post Sensitive Information

For your security, do not include:

- Passwords
- API keys
- Access tokens
- Full payment card information
- Full bank account information
- Private personal data that should not be public

If sensitive information needs to be shared, remove or redact it before posting.

## Response Expectations

We use GitHub Issues to organize and track support requests. Submitting an issue helps us:

- Review the request in one place
- Ask follow-up questions
- Track progress
- Keep a record of fixes and updates

## Thank You

Thank you for taking the time to submit a clear issue. It helps us support churches, nonprofits, and community organizations more effectively.
